# v0.4 Implementation Gate Checklist

## Gate A — Editor Model Gate

- [ ] Editor edits are represented as typed operations.
- [ ] Core rejects invalid operations.
- [ ] Operation replay produces deterministic document state.
- [ ] Undo/redo uses operation transactions, not UI snapshots.
- [ ] Decorations are derived from core diff state.

## Gate B — Merge Safety Gate

- [ ] Two-way merge remains available without base document.
- [ ] Three-way merge requires explicit base identification.
- [ ] Conflict IDs are stable across UI refresh.
- [ ] Unresolved conflict state blocks unsafe final save by default.
- [ ] Manual conflict edits are tracked as explicit resolutions.

## Gate C — File Safety Gate

- [ ] Loaded file snapshot records path, size, mtime, and content digest.
- [ ] Save checks current disk state before overwrite.
- [ ] External modifications trigger reconciliation UI.
- [ ] Backup policy is applied before destructive writes.
- [ ] Atomic write fallback behavior is documented per platform.

## Gate D — Scale Gate

- [ ] Directory comparison can be cancelled.
- [ ] Incremental refresh does not block the UI.
- [ ] Large-file mode is explicit and testable.
- [ ] Batch operations require preview.
- [ ] Patch apply requires preview.

## Gate E — v1 Governance Gate

- [ ] Public command IDs are frozen for v1.
- [ ] Session schema is versioned.
- [ ] Safety prompts are reviewed and stable.
- [ ] RFC acceptance criteria are mapped to tests.
- [ ] Deferred/non-goals are explicitly listed.
